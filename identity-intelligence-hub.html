CORS Proxy (opzionale ma consigliato)

La maggior parte dei provider OSINT (LeakIX, BreachDirectory, IntelX, DeHashed) non consente chiamate dirette dal browser per via delle policy CORS. Configura un proxy per inoltrare le richieste. Esempi pubblici: https://corsproxy.io/?, https://api.allorigins.win/raw?url=. Per uso reale, deploya un proxy tuo (Cloudflare Worker, Nginx, etc.).

Limite proxy pubblici: servizi come corsproxy.io spesso strippano gli header custom (x-key, Authorization, X-RapidAPI-Key) — quindi le API che usano questi header per autenticarsi falliscono comunque. Per usare davvero questa app deploya il tuo Cloudflare Worker (vedi sotto, 2 minuti gratis).

Anteprima URL chiamata: —
Esempi accettati: https://tuo-worker.workers.dev/ · https://tuo-worker.workers.dev/? · https://corsproxy.io/?. Il ? finale viene aggiunto automaticamente.

Verifica che il Worker inoltri correttamente — non richiede API Key

Deploy del tuo Cloudflare Worker (consigliato)

  1. Vai su dash.cloudflare.com → Workers & Pages → Create → Hello World.
  2. Copia il codice qui sotto e incollalo nell'editor del Worker, poi Deploy.
  3. Copia la URL che Cloudflare ti dà (es. https://intel-hub-proxy.tuo-account.workers.dev/?) e incollala nel campo proxy qui sopra.
const CORS = {
  'Access-Control-Allow-Origin': '*',
  'Access-Control-Allow-Methods': 'GET, POST, PUT, DELETE, OPTIONS',
  'Access-Control-Allow-Headers': '*',
  'Access-Control-Expose-Headers': '*',
  'Access-Control-Max-Age': '86400'
};

export default {
  async fetch(request) {
    // Preflight CORS: rispondi subito senza chiamare l'upstream
    if (request.method === 'OPTIONS') {
      return new Response(null, { status: 204, headers: CORS });
    }

    const url = new URL(request.url);
    const target = url.search.slice(1); // tutto dopo il "?"
    if (!target) {
      return new Response('Missing target URL. Usage: ' + url.origin + '/?https://api.example.com/path',
        { status: 400, headers: { ...CORS, 'Content-Type': 'text/plain' } });
    }

    let targetUrl;
    try { targetUrl = new URL(decodeURIComponent(target)); }
    catch { return new Response('Invalid target URL', { status: 400, headers: CORS }); }

    // Inoltra header del client, rimuovendo quelli che non vanno propagati
    const headers = new Headers(request.headers);
    ['host','origin','referer','cf-connecting-ip','cf-ipcountry','cf-ray','cf-visitor','x-forwarded-for','x-forwarded-proto','x-real-ip']
      .forEach(h => headers.delete(h));
    // Alcuni provider rifiutano richieste senza User-Agent
    if (!headers.has('user-agent')) headers.set('user-agent', 'IntelHub-Proxy/1.0');

    let upstream;
    try {
      upstream = await fetch(targetUrl.toString(), {
        method: request.method,
        headers,
        body: ['GET','HEAD'].includes(request.method) ? undefined : request.body,
        redirect: 'follow'
      });
    } catch (e) {
      return new Response('Upstream fetch failed: ' + e.message,
        { status: 502, headers: { ...CORS, 'Content-Type': 'text/plain' } });
    }

    const respHeaders = new Headers(upstream.headers);
    Object.entries(CORS).forEach(([k, v]) => respHeaders.set(k, v));
    return new Response(upstream.body, { status: upstream.status, headers: respHeaders });
  }
};
IntelX bloccato dai Cloudflare Workers? Alcuni provider OSINT (in particolare IntelX free tier) blackistano gli IP cloud: il Worker invia la richiesta ma upstream non risponde mai (timeout). Se vedi questo comportamento, deploya il proxy alternativo qui sotto su Render.com, che usa un range IP diverso.

Proxy alternativo: Node Express su Render.com (se il Worker viene bloccato)

  1. Vai su render.com → registrati gratis (no carta).
  2. New → Web Service → Public Git Repository.
  3. Crea una repo GitHub vuota e mettici 2 file: server.js (codice qui sotto) e package.json (con "dependencies": {"express": "^4.18.0", "node-fetch": "^2.7.0"}). Connettila a Render.
  4. Build command: npm install · Start command: node server.js · Plan: Free.
  5. Una volta deployato, copia la URL (es. https://intel-hub-proxy-xxxx.onrender.com) e incollala nel campo proxy qui sopra.
const express = require('express');
const fetch = require('node-fetch');
const app = express();

const CORS = {
  'Access-Control-Allow-Origin': '*',
  'Access-Control-Allow-Methods': 'GET, POST, PUT, DELETE, OPTIONS',
  'Access-Control-Allow-Headers': '*',
  'Access-Control-Expose-Headers': '*',
  'Access-Control-Max-Age': '86400'
};

app.use(express.raw({ type: '*/*', limit: '5mb' }));

app.options('*', (req, res) => res.set(CORS).status(204).end());

app.all('*', async (req, res) => {
  const target = req.url.slice(2); // rimuove '/?'
  if (!target) return res.set(CORS).status(400).send('Missing target URL');

  let targetUrl;
  try { targetUrl = new URL(decodeURIComponent(target)); }
  catch { return res.set(CORS).status(400).send('Invalid URL'); }

  const headers = { ...req.headers };
  ['host','connection','content-length','origin','referer','x-forwarded-for','x-forwarded-proto','x-forwarded-host','x-render-origin-server']
    .forEach(h => delete headers[h]);
  if (!headers['user-agent']) headers['user-agent'] = 'IntelHub-Proxy/1.0';

  try {
    const upstream = await fetch(targetUrl.toString(), {
      method: req.method,
      headers,
      body: ['GET','HEAD'].includes(req.method) ? undefined : req.body,
      redirect: 'follow',
      timeout: 25000
    });
    res.status(upstream.status);
    Object.entries(CORS).forEach(([k, v]) => res.set(k, v));
    upstream.headers.forEach((v, k) => {
      if (!['transfer-encoding','content-encoding'].includes(k.toLowerCase())) res.set(k, v);
    });
    upstream.body.pipe(res);
  } catch (e) {
    res.set(CORS).status(502).send('Upstream fetch failed: ' + e.message);
  }
});

const PORT = process.env.PORT || 3000;
app.listen(PORT, () => console.log('IntelHub proxy on ' + PORT));
